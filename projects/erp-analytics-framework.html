§ 01 — Overview

Project overview.

The ERP held everything — POs, MRs, GRNs, material issues, stock transfers, asset movements — but extracting clean, analysis-ready data was a specialist task repeated for every question. This framework standardises extraction so every report starts from the same reliable foundation.

DisciplineERP Reporting
PlatformePROMIS · JD Edwards · SQL
Timeline2017 — Present
ScopeInventory · Procurement · Assets
RoleAnalyst & Builder

Business Challenge

Getting usable data out of ePROMIS and JD Edwards was slow and inconsistent, and each new report reinvented the same extraction logic. The ERP functioned as a system of record but not as an analytics source.

The Solution

I built a reporting framework over both ERPs that standardises how transactions are pulled and structured, so inventory, procurement, asset, and operational analysis all begin from the same dependable base.
